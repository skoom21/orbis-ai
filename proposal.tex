\section*{Project Abstract }
\addcontentsline{toc}{section}{Project Abstract}

ORBIS AI is an \textbf{agentic travel planning and booking system} designed to function as a digital travel assistant. It provides users with an intelligent, conversational interface that simplifies complex travel planning workflows—searching and comparing flights, hotels, and activities across providers while generating personalized, day-by-day itineraries based on budget, preferences, and constraints.

The system leverages \textbf{agentic AI architecture}, where specialized agents (Planner, Flight, Hotel, Itinerary, Booking, and Verifier) collaborate to deliver accurate, transparent, and adaptable results. Users can initiate trip planning via natural language chat, receive curated travel bundles, refine itineraries interactively, and confirm reservations through a secure checkout process (sandbox/test mode supported).

Beyond initial booking, ORBIS AI provides real-time trip support such as transit guidance, activity reminders, and change management for flight/hotel modifications. Key features include Google OAuth authentication, preference learning via a cold-start questionnaire, calendar export (ICS), email notifications, and observability dashboards for tracing agent runs.

By combining \textbf{AI-driven orchestration, external travel APIs, and modern web technologies}, ORBIS AI demonstrates how agentic systems can bridge the gap between user intent and fragmented travel services. As a Final Year Project, it not only emphasizes technical implementation but also addresses usability, scalability, and ethical data handling challenges.

\section{Introduction }

Travel planning today is fragmented: users must visit multiple platforms for flights, hotels, and activities, manually reconcile prices and rules, and still risk missing optimal options. ORBIS AI addresses this by acting as an \textbf{autonomous travel agent}, capable of understanding natural language queries, breaking them into structured tasks, and integrating results from real APIs.

Objectives include:

\begin{itemize}
\item Develop a conversational web platform for travel planning.
\item Implement an \textbf{agentic AI architecture} with specialized agents for flights, hotels, itineraries, and bookings.
\item Integrate vendor APIs (Amadeus, Booking.com, Stripe sandbox, Google Maps) in test/demo environments.
\item Provide personalization through user profiles, embeddings, and preference learning.
\item Ensure security, observability, and compliance (OAuth, encryption, GDPR-style controls).
\end{itemize}

\section{Success Criterion}

Success is defined as a fully functional demo where:

\begin{enumerate}
\item A user signs in, initiates a trip request, and receives curated flight + hotel bundles.
\item The system generates a realistic, day-by-day itinerary within budget and preferences.
\item The user can hold options, perform a test payment, and receive booking confirmation with an ICS calendar export.
\item Change requests (e.g., shift departure date) are processed with updated options.
\end{enumerate}

If these workflows run end-to-end without errors under sandbox APIs, the project will be considered successful.

\section{Related work }

Previous AI-powered travel assistants, such as \textbf{Google Travel} and \textbf{Kayak AI chatbot}, provide partial solutions but lack deep conversational refinement or agent-based orchestration. Academic research highlights the use of \textbf{Retrieval-Augmented Generation (RAG)} for domain-specific planning and \textbf{multi-agent AI} for tool orchestration. Unlike single-model chatbots, ORBIS AI emphasizes \textbf{explainability, schema validation, and modular agents}, making it closer to research systems like LangChain Agents or Microsoft AutoGen but specialized for the travel domain.

\section{Project Rationale}

This project aligns with global demand for smarter, AI-driven personal assistants that reduce time and stress in travel planning. ORBIS AI demonstrates how \textbf{agentic AI can create real-world value}, while also giving the team practical experience in cloud deployment, API integration, and applied NLP—essential skills for both industry and research careers.

\section{Proposed Methodology and Architecture}

\textbf{Methodology:}

\begin{itemize}
\item \textbf{Agile development} with iterative sprints.
\item \textbf{Agentic AI approach} (Planner, Flight, Hotel, Itinerary, Booking, Verifier agents).
\item \textbf{RAG for personalization} (vector store with user preferences and static guides).
\item \textbf{Schema validation} to prevent hallucinations.
\item \textbf{Testing} via unit, contract, and E2E workflows.
\end{itemize}

\textbf{Architecture:}

\begin{itemize}
\item \textbf{Frontend:} Next.js (React, TS) hosted on Vercel.
\item \textbf{Backend:} Next.js API routes (or NestJS), deployed on Railway/Render.
\item \textbf{Database:} SurrealDB or Postgres; Redis for caching.
\item \textbf{APIs:} Amadeus, Booking.com, Stripe test, Google Maps.
\item \textbf{Observability:} Structured logs, agent run traces, error dashboards.
\end{itemize}
